# Contributing to Retail POS System

We love your input! We want to make contributing to this project as easy and transparent as possible.

## Development Process

1. Fork the repo and create your branch from main
2. Make your changes
3. Test thoroughly
4. Ensure code follows our standards
5. Submit a pull request

## Code Standards

### C# Coding Conventions
- Use PascalCase for class names and method names
- Use camelCase for local variables and method parameters
- Use meaningful, descriptive names
- Add XML comments for public methods

### Database Guidelines
- Use parameterized queries to prevent SQL injection
- Handle database exceptions appropriately
- Use transactions for multiple related operations

### UI/UX Standards
- Maintain consistent styling using UiStyles class
- Ensure forms are responsive and user-friendly
- Add proper validation and error messages

## Pull Request Process

1. Update README.md if needed
2. Test on multiple Windows versions if possible
3. Document new features
4. Get reviewed by maintainers

## Bug Reports

When filing an issue, please include:
- Windows version
- .NET version
- Steps to reproduce
- Expected vs actual behavior
- Screenshots if applicable
